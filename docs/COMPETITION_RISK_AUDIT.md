# 그늘로 경진대회 위험 감사

> 감사일: 2026-07-16
> 범위: 데이터 진실성, 검색·경로 신뢰성, 검증 주장, 발표 안정성, 개인정보

## 결론

손으로 만든 가짜 경로·건물·그림자 데이터는 확인되지 않았다. `web/mock`도 실데이터 산출물에서 생성한 정적 스냅샷이다. 공식 쉼터는 서울시 2026년 자료에서 만든 추적 산출물이다. 다만 현재 상태에서 “모든 상호 검색”, “실시간 실제 그림자”, “그늘이 기온을 낮춘다는 실증”, “폭염 보행의 의학적 안전 보장”을 주장하면 심사에서 반박될 수 있다.

## P0: 발표 전에 반드시 처리

| 위험 | 확인 근거 | 영향 | 조치 |
|---|---|---|---|
| S-DoT로 냉각 효과를 입증할 수 없음 | `r=0.092, p=0.246`; 시간 통제 회귀 계수 `+0.745`; 공간 CV 모델도 기준선 대비 평균 `-35.58%`, 개선 fold `0/5` | “실측 검증 완료”나 “AI 냉각 예측” 주장은 데이터와 반대 | 제안서에서 인과·냉각 입증 문구 제거. 모델은 게이트에 따라 제품에서 제외하고 보조 분석의 한계와 후속 현장실험 계획으로 제시 |
| 발표 네트워크 의존 | CARTO 타일, unpkg MapLibre, 선택적 Kakao 검색, Open-Meteo 현재 날씨 | 행사장 네트워크 장애 시 지도 타일·지도 엔진 로드와 외부 상호 검색이 실패하고 현재 날씨가 중립 폴백됨 | 발표 전 MapLibre 로컬 정적 자산과 오프라인 타일 범위를 준비하거나 화면 녹화·mock 스냅샷을 비상 시나리오로 준비. 날씨 장애는 경로·3D 동작을 막지 않음 |

## P1: 신뢰도에 직접 영향

| 위험 | 현재 상태 | 완화 방향 |
|---|---|---|
| 서비스 범위가 노원구뿐 | 노원구 밖 집 주소·현재 위치는 의도적으로 경로 그래프에 연결하지 않음 | 검색창과 제안서에 서비스 지역을 명시하고 발표 OD를 노원구 안으로 고정 |
| 그림자가 실시간처럼 보일 수 있음 | 대표일 2026-08-06, 맑은 하늘, 5개 경로 시간대 | UI는 `맑은 하늘 기준`으로 표기하고 구름·날씨 미반영과 계산 날짜는 데이터 방법론에 명시 |
| 현재 날씨가 공식 특보처럼 보일 수 있음 | Open-Meteo의 모델 현재값을 사용하고 체감온도 33°C 이상을 서비스 문구 트리거로 적용 | 기상청 폭염특보는 이틀 이상 지속 예상과 영향도 함께 판단한다. API에 `model_current_conditions`와 공급자를 보존하고 UI·발표에서 공식 기상청 특보라고 부르지 않음 |
| 건물 높이 품질 | 높이 결측 시 층수×3m, 둘 다 없으면 제외 | 원천 높이/대체/제외 비율을 파이프라인 산출 통계로 보존하고 표본 건물 현장 대조 |
| 그림자 기하 과대 추정 | 오목 건물도 convex hull로 채움 | 제안서에 1차 근사 명시. 고위험 구간은 ray casting 또는 건물별 union 비교 |
| 가로수 그늘 단순화 | 모든 가로수를 반경 3m 원으로 취급 | 수종·수관폭·계절·태양 방향 데이터가 없다는 점 명시. 민감도 분석 추가 |
| S-DoT 날짜/교란 불일치 | 관측은 6/15~7/5, 그림자는 8/6; 센서 36개만 격자 매칭 | 같은 날짜 그림자 재계산, 운량·토지피복 통제, 그늘/양지 쌍 센서 현장 설계 |
| OSM 보행망 스냅 | 출발·도착을 최근접 노드에 연결 | 건물 출입구·횡단보도·공사 정보 오차를 발표 전에 현장 보행으로 확인 |
| 모바일 실기기 성능 | Orca 390×844에서 전체 흐름·3D 저사양 모드·경계 상자·콘솔을 검증했다. 초기 전체 그림자 프레임 요청은 제거했지만 실제 저사양 Android/iOS는 미검증 | 발표 기기에서 10분 연속 실행·FPS·메모리·발열·WebGL 복구 테스트 |
| AI 활용 과장 | 제품 경로에는 LLM·비전·학습 모델 호출이 없고, 별도 S-DoT 모델은 공간 검증 게이트에 실패함 | “AI 경로 추천” 표현을 쓰지 않고 평가한 모델·탈락 판정·제품의 물리 계산을 구분 |
| 내부 일사 노출 파라미터의 오해 | 일반 20, 어르신 15 sun-minute는 경로 후보와 쉼터 탐색용 상대 비교 파라미터이며 임상·보건 기준이 아님 | 화면에는 임의 통과율 대신 예상 햇빛 노출 시간과 시뮬레이션 근거를 표시하고 `안전`, `위험 없음` 표현 금지. API 계약에는 모델·한계 메타데이터 유지 |
| 공식 쉼터 운영·접근성 | 297곳 중 일반 이용 후보는 38곳이고 259곳은 회원·특정계층 시설이다. 원천에 운영시간이 없음 | 자동 경유에는 `public`만 사용하고 모두 `운영 여부 확인 필요`로 표시. 제한 시설을 일반 이용 가능하다고 주장하지 않음 |
| 공식 쉼터 자료 시점 | 추적 산출물은 2026-07-16에 확인한 `OA-21065` 2026년 등록 자료이며 실시간 개방 상태가 아님 | `verified_at`, `source_year`, 원문 링크를 보존하고 발표 직전 `pipeline/13_cooling_spots.py`로 갱신 |

## P2: 운영·심사 질의 대비

| 위험 | 대응 |
|---|---|
| 검색어 개인정보 | 서버는 검색어를 DB에 저장하지 않지만 Kakao 활성화 시 검색어가 외부 공급자로 전송됨을 개인정보 처리 설명에 반영 |
| 데이터 라이선스/출처 | GIS건물통합정보, 서울시 가로수·S-DoT·무더위쉼터, OSM/CARTO의 출처와 라이선스를 제안서 및 서비스 하단에 정리 |
| mock 오해 | “가짜 데이터”가 아니라 실데이터 파생 발표 스냅샷임을 명시하고 생성 스크립트·생성일 기록 |
| 3D 표현 과장 | 현재는 높이 기반 MapLibre 압출 모델이며 사진 재질 디지털 트윈이 아님을 명시 |
| 접근성 | 키보드 검색, reduced-motion, 한국어 음성과 `role=status` 회전 안내를 지원하나 실제 장애인·스크린리더·색각 이상 사용자 검증은 남아 있음 |
| 음성 라이선스·기기 편차 | 상용 지도 앱 녹음 파일은 사용하지 않는다. 현재 Web Speech 음질은 기기 설치 음성에 따라 달라지므로 동일 품질이 필요하면 정식 TTS 계약과 실기기 평가가 필요 |

## 이번 감사에서 수정 완료

- 로컬 주소의 `서울`/`서울시`/`서울특별시` 표기와 공백 차이를 정규화했다.
- OSM 장소를 5,059곳으로 재색인하고 NaN 태그 때문에 전부 행정으로 분류되던 버그를 수정했다.
- 경로 그래프에서 누락됐던 12,430그루의 가로수 근사 그늘을 포함해 재계산했다.
- S-DoT 위치 XLSX, 실제 평균 기온·시간 컬럼, 밑줄 시간 형식을 읽도록 수정했다.
- UI 날짜 표현을 제거하고 `맑은 하늘 기준` 배지로 정리했다.
- 3D 건물 투명도를 제거해 뒤쪽 건물·그림자가 겹쳐 보이는 현상을 줄였다.
- Kakao Local을 활성화하고 `/api/health`의 `place_search.kakao=active`, 한국성서대학교·노원구청 도로명 검색을 확인했다.
- 진행 방향 부채꼴, 다음 회전 지도 마커, 거리 구간별 한국어 음성과 스크린리더 상태 안내를 추가했다.
- 현재 AI 모델이 구현되지 않았음을 프로젝트 문서와 위험 항목에 명시했다.
- S-DoT 상대 온도 모델을 센서 단위 GroupKFold로 평가하고 실패 결과를 숨기지 않은 채 제품 적용을 차단했다.
- 공식 무더위쉼터 297곳을 일반 이용 38곳과 제한 시설 259곳으로 분리하고, 경유 후보에는 일반 이용 시설만 노출한다.
- 사용자 화면을 예상 햇빛 노출 시간과 맑은 하늘 그림자 근거 중심으로 정리하고, 임의 한도·백분율은 API 내부 계산에만 유지한다.
- Orca 내부 브라우저에서 데스크톱 1138×871·모바일 390×844의 검색, 현재 날씨 안내, 경로, 1×~16× 미리보기, 내비게이션, 3D 라이트/다크를 확인하고 UI 겹침과 콘솔 오류가 없음을 검증했다.
- 장소 팝업을 330px 카드로 축소하고 모바일에서는 패널 아래 안전영역으로 자동 이동시키며 팝업이 열린 동안 겹치는 지도 도구를 숨긴다.
- 현재 체감온도 33°C 이상에서만 외출 미루기 문구가 나오며, 기준 미만 또는 Open-Meteo 장애에서는 중립 문구로 유지하도록 구현했다.
- 정적 경로·검색·3D 스냅샷과 키 없는 날씨 함수로 구성된 Vercel 배포 경로를 추가했다.
- 모바일·저사양 3D에서 초기 전체 시간 프레임 요청을 제거하고 선택 시간 단일 프레임만 지연 로드하도록 변경했다.
- 브라우저 응답 캐시 12개·서버 bbox 캐시 16개 상한, 노원구 밖·비정상 좌표 거부, Linux·Windows GitHub Actions 검증을 추가했다.

## 새 clone 재현성

| 위험 | 완화 조치 |
|---|---|
| 팀원·심사자가 원천 SHP/CSV를 받지 못하면 live 서버 실행을 재현하지 못할 수 있었다. | 재배포 가능한 실행 산출물 `data/processed` 8개(약 31.0MB)를 저장소에 포함한다. 원천 데이터와 `tmp`만 제외하고, 기본 `.env.example`은 PostGIS를 사용하지 않아 새 clone이 즉시 live API를 실행한다. |
| 행정안전부 전국 쉼터 전체 API는 신청·서비스 키가 필요해 새 clone 재현성을 해칠 수 있다. | 키 없이 전체 CSV를 내려받을 수 있는 서울시 공식 `OA-21065`를 원천으로 선택하고 58KB 처리 산출물을 함께 추적한다. 런타임 데이터와 API에는 추가 다운로드가 필요 없지만 지도 엔진·타일에는 네트워크가 필요하다. |
| 현재 날씨는 외부 서비스 장애·모델 오차가 있을 수 있다. | Open-Meteo는 키 없이 조회하고 5분 CDN/10분 애플리케이션 캐시로 호출을 제한한다. 실패 시 외출 자제 판단을 만들지 않고 중립 문구로 폴백하며 정적 검색·경로·3D는 계속 제공한다. |
