# 그늘로 경진대회 위험 감사

> 감사일: 2026-07-16
> 범위: 데이터 진실성, 검색·경로 신뢰성, 검증 주장, 발표 안정성, 개인정보

## 결론

손으로 만든 가짜 경로·건물·그림자 데이터는 확인되지 않았다. `web/mock`도 실데이터 산출물에서 생성한 정적 스냅샷이다. 다만 현재 상태에서 “모든 상호 검색”, “실시간 실제 그림자”, “그늘이 기온을 낮춘다는 실증”을 주장하면 심사에서 반박될 수 있다.

## P0: 발표 전에 반드시 처리

| 위험 | 확인 근거 | 영향 | 조치 |
|---|---|---|---|
| S-DoT로 냉각 효과를 입증할 수 없음 | `r=0.092, p=0.246`; 시간 통제 회귀 계수 `+0.745` | “실측 검증 완료” 주장은 데이터와 반대 | 제안서에서 인과·냉각 입증 문구 제거. 보조 분석의 한계와 후속 현장실험 계획으로 제시 |
| 발표 네트워크 의존 | CARTO 타일, unpkg MapLibre, jsDelivr 폰트, Kakao 검색 | 행사장 네트워크 장애 시 지도/아이콘/검색 실패 | 발표 전 로컬 정적 자산과 오프라인 타일 범위를 준비하거나 화면 녹화·mock 스냅샷을 비상 시나리오로 준비 |

## P1: 신뢰도에 직접 영향

| 위험 | 현재 상태 | 완화 방향 |
|---|---|---|
| 서비스 범위가 노원구뿐 | 노원구 밖 집 주소·현재 위치는 의도적으로 경로 그래프에 연결하지 않음 | 검색창과 제안서에 서비스 지역을 명시하고 발표 OD를 노원구 안으로 고정 |
| 그림자가 실시간처럼 보일 수 있음 | 대표일 2026-08-06, 맑은 하늘, 5개 경로 시간대 | UI는 “시뮬레이션”으로 표기. 구름·날씨 미반영을 데이터 방법론에 명시 |
| 건물 높이 품질 | 높이 결측 시 층수×3m, 둘 다 없으면 제외 | 원천 높이/대체/제외 비율을 파이프라인 산출 통계로 보존하고 표본 건물 현장 대조 |
| 그림자 기하 과대 추정 | 오목 건물도 convex hull로 채움 | 제안서에 1차 근사 명시. 고위험 구간은 ray casting 또는 건물별 union 비교 |
| 가로수 그늘 단순화 | 모든 가로수를 반경 3m 원으로 취급 | 수종·수관폭·계절·태양 방향 데이터가 없다는 점 명시. 민감도 분석 추가 |
| S-DoT 날짜/교란 불일치 | 관측은 6/15~7/5, 그림자는 8/6; 센서 36개만 격자 매칭 | 같은 날짜 그림자 재계산, 운량·토지피복 통제, 그늘/양지 쌍 센서 현장 설계 |
| OSM 보행망 스냅 | 출발·도착을 최근접 노드에 연결 | 건물 출입구·횡단보도·공사 정보 오차를 발표 전에 현장 보행으로 확인 |
| 모바일 실기기 성능 | 390×844 Orca 뷰포트와 3D 점진 로딩은 검증 완료, 실제 저사양 Android/iOS 미검증 | 발표 기기에서 10분 연속 실행, FPS·메모리·발열·WebGL 복구 테스트 |
| AI 활용 과장 | 현재 제품 경로에는 실제 LLM·비전·학습 모델 호출이 없고 물리 계산·규칙 템플릿·기기 TTS만 있음 | 실제 모델과 평가 결과 전에는 “AI 경로 추천” 표현을 쓰지 않고 구현/계획을 구분 |

## P2: 운영·심사 질의 대비

| 위험 | 대응 |
|---|---|
| 검색어 개인정보 | 서버는 검색어를 DB에 저장하지 않지만 Kakao 활성화 시 검색어가 외부 공급자로 전송됨을 개인정보 처리 설명에 반영 |
| 데이터 라이선스/출처 | GIS건물통합정보, 서울시 가로수·S-DoT, OSM/CARTO의 출처와 라이선스를 제안서 및 서비스 하단에 정리 |
| mock 오해 | “가짜 데이터”가 아니라 실데이터 파생 발표 스냅샷임을 명시하고 생성 스크립트·생성일 기록 |
| 3D 표현 과장 | 현재는 높이 기반 MapLibre 압출 모델이며 사진 재질 디지털 트윈이 아님을 명시 |
| 접근성 | 키보드 검색, reduced-motion, 한국어 음성과 `role=status` 회전 안내를 지원하나 실제 장애인·스크린리더·색각 이상 사용자 검증은 남아 있음 |
| 음성 라이선스·기기 편차 | 상용 지도 앱 녹음 파일은 사용하지 않는다. 현재 Web Speech 음질은 기기 설치 음성에 따라 달라지므로 동일 품질이 필요하면 정식 TTS 계약과 실기기 평가가 필요 |

## 이번 감사에서 수정 완료

- 로컬 주소의 `서울`/`서울시`/`서울특별시` 표기와 공백 차이를 정규화했다.
- OSM 장소를 5,059곳으로 재색인하고 NaN 태그 때문에 전부 행정으로 분류되던 버그를 수정했다.
- 경로 그래프에서 누락됐던 12,430그루의 가로수 근사 그늘을 포함해 재계산했다.
- S-DoT 위치 XLSX, 실제 평균 기온·시간 컬럼, 밑줄 시간 형식을 읽도록 수정했다.
- UI 날짜 배지를 `8월 6일 시뮬레이션`으로 수정했다.
- 3D 건물 투명도를 제거해 뒤쪽 건물·그림자가 겹쳐 보이는 현상을 줄였다.
- Kakao Local을 활성화하고 `/api/health`의 `place_search.kakao=active`, 한국성서대학교·노원구청 도로명 검색을 확인했다.
- 진행 방향 부채꼴, 다음 회전 지도 마커, 거리 구간별 한국어 음성과 스크린리더 상태 안내를 추가했다.
- 현재 AI 모델이 구현되지 않았음을 프로젝트 문서와 위험 항목에 명시했다.
