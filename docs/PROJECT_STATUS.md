## 1. 서비스 범위

그늘로는 노원구 안에서 출발지와 목적지를 검색하고, 2026-08-06의 태양 위치와 건물 높이·가로수를 이용해 햇빛 노출이 적은 도보 경로를 비교하는 시민용 내비게이션이다. 출발지의 현재 기온·체감온도를 먼저 확인하고, 현재 체감온도가 33°C 이상일 때만 외출을 미루는 문구를 표시한다. 그 밖에는 일반/어르신 동행별 예상 햇빛 노출 시간과 공식 무더위쉼터 후보를 중립적으로 비교한다.

- 서비스 지역: 노원구 보행망. 노원구 밖 주소와 경로는 지원하지 않는다.
- 그림자 의미: 맑은 하늘을 가정한 물리 시뮬레이션. 실시간 관측 그림자가 아니다.
- 행정용 SDI Top-10 분석 파이프라인과 보고 산출물은 존재하지만 현재 시민용 경로 화면에는 연결하지 않는다.

## 2. 구현된 사용자 흐름

1. 검색 화면에서 상호명, 건물명, 행정구역, 지번 주소를 출발지·목적지로 지정한다.
   - 장소를 고르면 지도 카메라가 해당 지점으로 먼저 이동해 위치를 확인한 뒤 다음 입력 또는 경로 계산을 시작한다.
   - `다시 입력`과 `출발·도착 바꾸기` 명령으로 경로 조건을 즉시 수정할 수 있다.
2. 두 지점이 정해지면 출발지의 현재 기온·체감온도와 물·양산·모자, 햇빛 노출이 적은 시간·경로 비교 안내를 표시한다.
   - 현재 체감온도 33°C 이상에서만 `지금은 외출을 미루는 편이 좋습니다`를 표시한다.
   - 현재값이 기준 미만이거나 외부 날씨 공급자가 응답하지 않으면 외출 자제 문구를 표시하지 않고 중립 안내로 유지한다.
3. 사용자가 이동 경로 비교를 선택하면 일반 또는 어르신 동행 모드로 최대 5개 경로의 시간·거리·그늘 비율·예상 햇빛 노출 시간을 비교한다.
   - 내부 노출 관리 파라미터를 넘는 경로는 기준 도달 전 경로 가까이에 있는 일반 이용 가능 공식 무더위쉼터를 최대 2곳까지 후보로 표시한다.
   - 운영시간 미제공 시설을 현재 운영 중이라고 표시하지 않고, 방문 전 확인 필요와 서울 열린데이터광장 출처를 함께 보여준다.
4. 경로를 고르면 기본적으로 GPS 위치를 추적하는 도보 길찾기 화면으로 전환되고 진행 방향 부채꼴, 다음 회전, 남은 거리, 주변 장소 라벨이 움직임을 따라간다.
   - `경로 미리보기`를 눌렀을 때만 시뮬레이션 러너가 자동 이동하며, 1×·2×·4×·8×·16× 중 속도를 바꾸거나 실시간 안내에 즉시 복귀할 수 있다.
   - 카메라 중심·확대·피치·방위각은 짧은 보간 루프로 연결해 교차로에서 시점이 급회전하지 않는다.
   - 다음 좌·우회전 지점은 지도 위 아이콘으로 표시하고, 거리 구간별 한국어 음성과 스크린리더 상태 메시지를 한 번씩 안내한다.
5. 모바일에서 지도를 드래그하거나 회전하면 추적을 잠시 멈추고, 재중심 버튼 또는 5초 후 자동으로 경로 중심을 다시 따른다.
6. 라이트/다크 모드, 시간별 그림자 재생, 별도 3D 근거 화면을 제공한다.

지도 클릭도 출발·도착 지정의 보조 수단으로 유지하지만 검색이 기본 흐름이다. 출발·도착 교환과 각 필드 초기화는 언제든 가능하다.

- 로컬 주소 검색은 `서울특별시`/`서울시`/`서울`과 공백·구분자를 정규화하고, 여러 단어가 필드에 흩어져 있어도 검색한다.
- OSM 색인을 2026-07-15 재생성해 상점·음식점·카페·체육시설 분류를 복구했다.
- Kakao Local은 2026-07-15 실호출에서 `active`이며 도로명 주소와 세부 상호를 반환한다. 한국성서대학교·노원구청 검색을 브라우저에서 확인했다.
- 외부 검색이 실패하면 API는 24,466건 로컬 색인으로 폴백하고 `/api/health`에 공급자 상태와 안전한 오류 코드만 노출한다.
- 공식 설정 문서: https://developers.kakao.com/docs/ko/local/common

## 7. 계산과 mock의 의미

- 계산 시간대: 10, 13, 14, 15, 17시. 그 사이 시각은 엣지 그늘율을 선형 보간한다.
- 3D 화면용 그림자는 07~18시 정시 프레임을 교차 페이드한다.
- 3D 진입 시 건물·경로·현재 시각 단일 프레임을 먼저 병렬 로드한다. 표준 모드는 전체 12시간 프레임을 유휴 시간에 준비하고, 모바일·저사양 모드는 선택한 시간 프레임만 필요할 때 가져온다.
- 건물·그림자·태양 위치 API는 같은 bbox 재방문 시 브라우저 최대 12개, 서버 bbox 최대 16개 범위에서 최대 1시간 재사용한다.
- 정적 배포용 건물 스냅샷도 노원구 전체 12,273동을 포함한다. 표준/저사양 단순화 파일을 분리하고 각 피처의 bbox로 현재 화면과 겹치는 건물만 MapLibre에 전달한다.
- 건물 그림자는 footprint와 이동 footprint의 convex hull 근사다.
- 가로수는 수종·수관·태양 방향과 무관한 반경 3m 원으로 근사한다.
- `web/mock`은 가짜 좌표를 손으로 만든 데이터가 아니다. `pipeline/10_mockdata.py`가 현재 실데이터 파이프라인 산출물에서 뽑은 정적 발표용 스냅샷이며, 건물은 데모 경로 주변으로 자르지 않고 노원구 전체를 보존한다.
- `?mock=1` 또는 정적 서버 기본 조건에서만 mock을 사용하고, FastAPI 8000 포트는 live API를 사용한다.
- 현재 날씨는 mock으로 만들지 않는다. 로컬 FastAPI와 Vercel 함수 모두 키 없는 Open-Meteo 현재 모델값을 사용하며, 공급자 장애 시 UI는 중립 상태로 폴백한다.
- 33°C는 기상청 폭염주의보의 체감온도 수준을 현재값에 적용한 서비스 트리거다. 공식 특보는 이틀 이상 지속 예상과 영향도 함께 판단하므로 앱 응답을 공식 기상청 특보라고 부르지 않는다.
- `sun_exposure_proxy_v1`은 경로를 걷는 동안의 시간 진행을 반영해 엣지 이동시간 × 햇빛 비율을 적산하는 상대 비교용 물리 근사다.
- 일반 20 sun-minute, 어르신 동행 15 sun-minute는 API 내부의 상대 비교 파라미터다. 화면은 임의 기준 통과 여부 대신 예상 햇빛 노출 시간과 `맑은 하늘 그림자 기반 예상치`를 표시한다.

## 8. AI 활용 상태

현재 실제 제품 경로에는 LLM·비전 모델·학습 모델 호출이 없다. S-DoT 학습 모델은 별도 평가했지만 사전 게이트를 통과하지 못해 제품 산출물과 경로 계산에서 제외했다.

- 경로 추천은 태양·건물·가로수 물리 시뮬레이션과 가중 최단경로 계산이다.
- 경로 인사이트는 수치 조건에 따른 규칙 기반 템플릿이다.
- 음성 안내는 브라우저 Web Speech API와 기기 한국어 음성을 사용하며 앱이 운영하는 AI 모델이 아니다.
- `UI_PLAN.md`의 `/api/brief`, `/api/parse_query`, 거리뷰 비전 검증은 계획이며 아직 구현되지 않았다.
- `sdot_relative_temperature_hgb_v1`은 센서 단위 GroupKFold로 실행·평가했으며 기준선보다 악화되어 `eligible_for_product=false`다.
- LightGBM·SHAP은 선택 의존성이며 현재 제품 실행에 포함되지 않는다.

따라서 현재 기능을 “AI 경로 추천”으로 표현하지 않는다. 발표에서는 “AI 가능성을 정량 평가했으나 성능 게이트에 따라 채택하지 않은 물리 기반 경로 서비스”로 구분한다.

Vercel 프로덕션 `https://gneulro.vercel.app`에서도 공릉 2D 데스크톱·모바일과 저사양 3D를 확인했으며, 정적 건물은 전체 12,273동·공릉 점검 bbox 2,807동으로 응답했다. 해당 3D 세션에서 저사양 건물 파일은 압축 전송 553KB, 응답 258ms, 로드 후 JS heap 17MB로 관측됐고 콘솔 오류는 없었다. 이 수치는 검증 환경의 1회 관측값이며 실제 저사양 Android/iOS의 장시간 발열·메모리 검증은 남아 있다.

데이터 재생성 순서는 `pipeline/01_prepare.py`부터 `04_graph.py`까지이며, 검색은 `08_places.py`, 그림자 애니메이션은 `09_shadow_frames.py`, 정적 스냅샷은 `10_mockdata.py`다. S-DoT 보조 분석은 `05_validate.py`, 모델 게이트는 `12_microclimate.py`, 공식 쉼터는 `13_cooling_spots.py`다. 새 clone 실행에는 추적된 산출물을 사용하므로 원천 데이터나 파이프라인 재실행이 필요 없다.

## 13. 새 clone 실행 보장

- 실행에 필요한 `data/processed` GeoParquet·GraphML·장소 색인을 저장소에 포함한다. 새 clone은 제외된 원천 SHP/CSV·PostGIS·Kakao 키·무더위쉼터 API 키 없이도 기본 FastAPI 실행에서 live API를 사용한다.
- `data/raw`와 `tmp`는 Git에서 제외한다. 공식 쉼터 재생성은 키 없는 `pipeline/13_cooling_spots.py`로 가능하지만, 기본 실행은 추적된 약 58KB 산출물을 바로 읽는다.
- `web/mock`은 정적 서버·오프라인 발표용 실제 파이프라인 스냅샷이며, FastAPI 8000 포트에서는 live API가 기본값이다. 파이프라인 산출물이 없는 특수 환경에서만 `/api/health`가 `status: demo`, `data_mode: snapshot`을 반환한다.
- Vercel 배포는 `web/mock` 파이프라인 스냅샷으로 검색·경로·3D를 제공하고 `/api/weather`만 경량 함수로 실시간 조회한다. Open-Meteo 또는 네트워크 장애 시 날씨 문구만 중립 폴백하며 나머지 기능은 계속 동작한다.
- Git 추적 파일은 약 39.3MB이고 최대 단일 파일은 약 10.5MB로 GitHub의 파일당 100MB 제한 안에 있다. GitHub Actions가 Ubuntu/Python 3.11과 Windows/Python 3.12에서 기본 실행 의존성과 필수 산출물을 검사한다.
