# 그늘로 저장소 작업 규칙

## 상시 문서 갱신

- 코드, API 계약, 화면 흐름, 데이터 파이프라인, 산출물 또는 실행 방법을 바꾼 작업은 종료 전에 반드시 `docs/PROJECT_STATUS.md`를 함께 갱신한다.
- 위험, 한계, 외부 서비스 의존성 또는 검증 결과가 달라지면 `docs/COMPETITION_RISK_AUDIT.md`도 갱신한다.
- `docs/PROJECT_STATUS.md`의 `최종 갱신`, 관련 섹션, 변경 기록 세 곳을 확인한다. 구현되지 않은 기능을 완료로 적지 않는다.
- 관측값, 공공 원천 데이터, 물리 시뮬레이션, 원천 데이터에서 추출한 mock 스냅샷을 문서와 UI에서 서로 구분한다.

## 프론트엔드

- 지도 컨트롤, 패널, 안전영역과 새 UI가 데스크톱 및 모바일에서 겹치지 않는지 실제 브라우저로 검증한다.
- 지도 중심을 가려야 하는 고정 UI는 만들지 않는다. 모바일 내비게이션은 경로와 현재 위치가 가장 먼저 보여야 한다.
- 애니메이션은 `transform`과 `opacity` 중심으로 구현하고 `prefers-reduced-motion`을 존중한다.
- 변경 후 최소한 데스크톱 1개, 모바일 1개 뷰포트에서 스크린샷과 콘솔 오류를 확인한다.

## 데이터와 주장

- 경로 서비스 범위는 노원구 보행망이다. 노원구 밖 위치를 지원한다고 주장하지 않는다.
- 2026-08-06 그림자는 맑은 하늘 가정의 물리 시뮬레이션이며 실측 그림자가 아니다.
- S-DoT 분석은 보조 검증이다. 현재 결과로 냉각 효과나 인과관계를 주장하지 않는다.
- fake 데이터는 추가하지 않는다. `web/mock`은 파이프라인 산출물에서 `pipeline/10_mockdata.py`로 생성한다.

## 완료 기준

- Python: `.venv\Scripts\python.exe -m ruff check .`와 `.venv\Scripts\python.exe -m pytest -q`
- JavaScript: `web/js/*.js`에 대해 `node --check`
- 서버: `/api/health`, 장소 검색, 경로 선택, 내비게이션, 3D 라이트/다크 화면 확인
